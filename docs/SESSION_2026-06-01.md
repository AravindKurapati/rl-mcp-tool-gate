# Session log: 2026-06-01

What shipped, what the numbers say, and what is left.

## What shipped

**Two PRs, both green, both targeting `main`.**

### PR #3 - gate decision logger
`feat(proxy): gate decision logger`

The proxy now writes every `tools/list` interception and every `tools/call`
to a local SQLite store at `~/.tool_gate/decisions.db`. Each call carries
`was_surfaced`: was the tool the agent invoked in the most recent gated
set? That single boolean is the reward signal a future contextual bandit
would learn from, and it is also a cleaner real-traffic eval than the
current `tools_called`-only ground truth.

Key design choices:
- Async write queue so the proxy hot path never blocks on disk
- Logger swallows write errors so it can never break the proxy
- Session ID derived from per-boot UUID, so one Claude Code window equals one session
- Can be disabled with `enable_decision_log = false` in `upstreams.toml`
- New module `src/eval/decision_log.py` for offline analysis, including a
  `bandit_dataset()` materializer that pairs decisions with their downstream calls
- New `SCHEMA.md` documents both tables
- 11 new tests

Specs that landed alongside the code:
- `FEATURE_gate_decision_log.md`
- `FEATURE_ground_truth_v2.md`

### PR #4 - heuristic ground-truth filters
`feat(eval): heuristic ground-truth filters for afr extraction`

Pass 1 of `FEATURE_ground_truth_v2.md`. Adds three filter options to
`src/eval/afr_extract.py`:

- `drop_failed`: skip `tool_calls` rows with `status='error'`
- `dedup_window_s`: collapse same-tool retries within N seconds
- `min_tools`: drop runs left empty after filtering

`--filtered` shortcut enables the recommended set (`drop_failed=True`,
`dedup_window_s=30`) and writes to `sessions_filtered.jsonl`.

Also fixed a long-standing label bug where SFT checkpoints were reported
as `bge_rl` in result JSON. Method is now inferred from the checkpoint
directory name.

9 new tests, full suite 60 green.

## What the numbers say

The afr DB grew from 99 to 138 usable runs since May; MCP-using runs grew
from 26 to 32. SFT checkpoint unchanged.

**MCP-domain recall@k on n=32 real runs**

| k | Off-the-shelf noisy | Off-the-shelf filtered | SFT noisy | SFT filtered |
|---|:---:|:---:|:---:|:---:|
| 3 | 0.102 | 0.117 | 0.234 | 0.219 |
| 5 | 0.184 | 0.169 | 0.281 | 0.273 |
| 8 | 0.291 | 0.307 | 0.464 | 0.448 |
| 12 | 0.442 | 0.437 | 0.589 | 0.573 |
| 20 | 0.517 | 0.512 | 0.724 | 0.740 |

**Three takeaways:**

1. **The SFT win held up at the larger n.** Recall@8 stayed in the 0.45 to
   0.46 band. Non-breaking@8 still 0.344. The May numbers were not a
   small-sample fluke.

2. **The filter barely changed anything.** Within ~1pp in either direction.
   The retry-inflation hypothesis was wrong: most retries in real Claude
   Code traffic are built-ins (Read, Bash, Edit), not MCP tools. The
   MCP-only eval was already clean.

3. **The off-the-shelf baseline improved on the larger sample (0.223 to 0.291
   at k=8).** SFT's lead narrowed from ~23pp to ~16pp. Still a clear win,
   multiplier closer to 1.5x than 2x. The honest story moves with the data.

Writeup: `docs/RESULT_filters_impact.md`.

## What this means for the portfolio

The May framing was: SFT roughly doubles off-the-shelf on real traffic.
That is no longer the exact number to quote. The new framing:

> "I trained an RL gate, built real-traffic eval that exposed it overfit,
> then proved a simpler SFT baseline keeps a 15-point lead in recall and
> a 12-point lead in non-breaking rate against off-the-shelf retrieval -
> on the full 32 real Claude Code sessions I have logged so far. The
> result is robust to noise filters on the labels, which was the first
> thing I checked after the v1 overfit."

That is a stronger story, not a weaker one: it shows iteration discipline.

## What is left

- **Merge the two open PRs.** Order does not matter; they touch different files.
- **Keep using the proxy.** The decision logger is now writing every gate
  decision to `~/.tool_gate/decisions.db`. Every session you log makes the
  next eval round stronger and gets you closer to enough data for the
  bandit pivot.
- **Pass 2 of ground-truth v2 (LLM gold labeling, ~$15)** - deprioritized.
  Was supposed to defend against "your eval is noise"; Pass 1 already
  showed that is not the failure mode here. Worth doing as eval-rigor
  upgrade for the next model round, not as a fix.
- **Bigger catalog.** When you add a few more MCP servers (Slack, Stripe,
  Notion) and reach ~200 tools, rerun the whole eval. SFT's lead should
  widen because retrieval baselines degrade faster on long-tail tools.
- **Bandit pivot** (`docs/FUTURE_bandit_pivot.md`). Unblocked once PR #3
  merges. Not urgent; baseline to beat is now SFT, not off-the-shelf.
